# TypeScript 前端基础学习计划

## 学习定位

你已经有其他语言经验，所以这次不需要按“纯零基础编程”节奏学。重点应该放在三件事上：

1. 补齐前端开发真正常用的 JavaScript。
2. 用 TypeScript 建立更清晰的类型思维。
3. 为后续 `React + TypeScript + 接口联调` 打基础。

这份计划不追求一次学全，而是追求能尽快开始写前端代码。

## 总体目标

完成这一轮后，你应该能够：

- 看懂并书写现代 JavaScript 语法。
- 理解 TypeScript 的常用类型写法。
- 写出带类型的函数、对象、数组和异步请求代码。
- 读懂常见前端项目里的 `tsconfig.json`、`import/export` 和接口类型。
- 为后续学习 React 组件、状态和接口请求做好准备。

## 建议节奏

建议每次学习 60 到 90 分钟：

- 15 分钟：看概念和示例。
- 30 到 40 分钟：手写练习。
- 15 到 20 分钟：运行代码、观察输出。
- 10 分钟：对比你熟悉的语言，记差异点。

不要只看，尽量每次都运行至少一个 `.ts` 文件。

## 学习顺序

### 第 1 阶段：先补 JavaScript 核心

先学“TypeScript 依赖的 JavaScript”，而不是试图一次补完所有前端知识。

重点：

- `let` 和 `const`
- 基本类型和值
- 函数与箭头函数
- 对象与数组
- 解构赋值
- 展开运算符
- 模块化 `import/export`
- `map`、`filter`、`find`
- `Promise`
- `async/await`

目标：

- 看懂现代前端代码最常见的语法。
- 知道哪些写法是 JavaScript 本体，哪些是 TypeScript 增强。

### 第 2 阶段：TypeScript 核心

这一阶段是主线。

重点：

- 基本类型标注
- 函数参数和返回值类型
- 对象类型
- `type`
- `interface`
- 可选属性
- 联合类型
- 字面量类型
- 类型收窄
- 泛型基础
- `unknown` 和 `any`

目标：

- 写简单业务代码时，能自然地补上类型，而不是把类型当成额外负担。

### 第 3 阶段：前端场景里的 TypeScript

重点：

- 表单数据类型
- API 返回值类型
- 列表渲染数据类型
- 用户信息、商品信息等对象建模
- `fetch` 请求和异步错误处理
- 基础 DOM 事件类型

目标：

- 开始把 TypeScript 用在真实前端场景，而不是只停留在语法题。

### 第 4 阶段：React + TypeScript 入门

这部分建议等当前目录的基础练习做完再继续。

重点：

- 组件 `props` 类型
- `useState` 类型
- 事件类型
- 接口数据渲染
- 可复用组件的泛型基础

如果你只有一周时间，优先拿下这些能力：

## 学习方法建议

- 每次只学一个文件，不要贪多。
- 先在聊天里看示例和解释，再手动抄写到本地。
- 抄完先运行，再自己改一遍。
- 每个文件底部的练习尽量不要直接看答案。
- 遇到不理解的地方，优先问“这段代码在前端里通常解决什么问题”。
